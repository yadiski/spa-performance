## Summary of Material Gaps

The following gaps represent the highest-priority items to address before a formal audit:

1. **No formal ISMS policy (A.5.1):** Draft a concise information security policy approved by management.
2. **MFA not mandated (A.9.5):** Enforce TOTP for `hra` and `it_admin` roles before go-live.
3. **No dependency vulnerability scanning (A.8.25):** Add `bun audit` or Dependabot to CI.
4. **No penetration test (A.14.2):** Commission an external pen test covering auth, RBAC bypass, injection, and file access.
5. **No DPIA for PDPA (A.18.1):** Conduct a Data Protection Impact Assessment for the performance appraisal data.
6. **No formal RTO/RPO agreement (A.17.1):** Agree recovery time and recovery point objectives with business stakeholders.
7. **Audit log hash not externally anchored (A.12.2):** Store a daily hash checkpoint outside the DB (e.g., in a separate audit-only store) to strengthen tamper evidence.
